## 初期仕様：Pogostuck Loot Mode オートスプリッター

### 背景
- ゲーム：Pogostuck（3D Gamestudio A8製、PC版）
- 目的：Loot ModeのLevel1〜10のラップタイムをLiveSplitで自動記録し、OBSで配信表示
- ログ経由の連携は不可のため、画面OCRで実装済み

### 現在の実装概要
- mss で画面左上の小領域（250×30px）を0.5秒ごとキャプチャ
- numpy で黄色ピクセル（R>180, G>150, B<80）だけ白に変換
- Tesseract OCR で文字認識（whitelist: 0123456789@|）
- 正規表現で「@X|」のXをレベル番号として抽出
- 同じ値が2回連続で出たら確定（安定化フィルタ）
- LiveSplit Server（TCP 16834）に reset/starttimer/split を送信

### 画面上の文字フォーマット
1)三つ編みひなみ_てっちゃん @ 1 | 0
                              ↑レベル ↑スコア（コイン枚数）

### 追加したい機能
「| 0」つまりスコアが0に戻ったことを検知して
Level1プレイ中のゲームリセットを検知する。
現状はlast_split_level > 1 からLevel1に戻った時しかリセット検知できない。

### 0.5秒ごとのフロー
┌───────────────────────────────┐  
│  ① 画面の左上 250×30px だけをキャプチャ              │  
│     （全画面でなく小領域なので負荷ほぼゼロ）             │  
└────────────────┬──────────────┘  
                 ↓  
┌───────────────────────────────┐  
│  ② numpy で黄色ピクセルだけを白に変換                  │  
│     R>180 かつ G>150 かつ B<80 → 白                 │  
│     それ以外（背景・背景が緑でも青でも）→ 黒             │  
│                                                    │  
│  変換前: calibrate2.pyのような黄文字+色背景            │  
│  変換後: 黒背景に白文字だけの画像                       │  
└────────────────┬──────────────┘  
                 ↓  
┌───────────────────────────────┐  
│  ③ 画像を3倍に拡大                                   │  
│     小さい文字はOCRが苦手なので引き伸ばして精度UP         │  
└────────────────┬──────────────┘  
                 ↓  
┌───────────────────────────────┐  
│  ④ Tesseract OCR で文字を読み取る                    │  
│     読める文字を「0123456789@|」に限定                 │  
│     → ノイズを減らし誤認識を抑制                       │  
│                                                    │  
│  例: '17@2|488'  ← こんな文字列が返ってくる            │  
└────────────────┬──────────────┘  
                 ↓  
┌───────────────────────────────┐  
│  ⑤ 正規表現でレベル番号を抽出                         │  
│                                                   │  
│  パターンA: @ の直後の1〜2桁 + |                      │  
│    '17@2|488' → @ の後 → 2                        │  
│                                                   │  
│  パターンB（@が消えた場合のfallback）:                │  
│    数字 + | + 数字 の最初の1〜2桁                    │  
│    '1723|963' → 3                                 │  
└────────────────┬──────────────┘  
                 ↓  
┌───────────────────────────────┐  
│  ⑥ 安定化フィルタ（STABLE_COUNT=2）                  │  
│                                                    │  
│  同じ値が2回連続で出た場合のみ「確定」                   │  
│  1回だけの誤認識はスルーする                           │  
│                                                    │  
│  例: '2' '2' → ✅ 確定                             │  
│      '2' '7' '2' → 7は1回だけなのでスルー            │  
└────────────────┬──────────────┘  
                 ↓  
┌───────────────────────────────┐  
│  ⑦ レベル変化に応じてLiveSplitに命令送信              │  
│                                                   │  
│  初めてLevel1を確認 → reset + starttimer           │  
│  レベルが上がった   → split                         │  
│  Level1に戻った     → reset + starttimer（リセット） │  
└────────────────┬──────────────┘  
                 ↓  
┌───────────────────────────────┐  
│  ⑧ LiveSplit がタイムを記録・表示                    │  
│     OBS がウィンドウキャプチャで配信に映す              │  
└───────────────────────────────┘  

### リセット挙動の捕捉
Level1のまま手動リセット → スクリプトはLevel1のままと認識
                          → last_split_level は 1 のまま
                          → 「Level1に戻った（>1からの降下）」ではないのでスルー
                          → タイマーは動き続ける ✅ 正しい把握
